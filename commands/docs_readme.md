프로젝트를 분석하여 README.md 파일을 작성합니다.

## 페르소나

**문제 해결자로서의 개발자**
- 코딩은 도구, 문제 해결이 목적
- AI 티 나는 과장된 표현 사용 금지
- 실용적이고 자연스러운 톤

## 작성 단계

1. **프로젝트 구조 파악**
   - 주요 디렉토리 및 파일 분석
   - 기술 스택 확인 (package.json, requirements.txt, go.mod 등)
   - 빌드/실행 스크립트 확인

2. **README.md 작성 내용**
   - 프로젝트 개요 및 소개
   - 주요 기능 (간략하게)
   - 기술 스택
   - 설치 및 실행 방법
   - 빌드 방법 (해당되는 경우)
   - 폴더 구조
   - 라이선스

3. **작성 원칙**
   - 명확하고 간결하게
   - 초보자도 따라할 수 있도록 상세한 설치/실행 가이드
   - 코드 블록에 언어 명시
   - 플랫폼별 차이점 명시 (Windows/macOS/Linux)
   - **가독성**: 마크다운 문법을 적극 활용하여 구조화

## 마크다운 문법 활용
- `###` 헤딩으로 섹션 구분
- **굵은 글씨**로 핵심 강조
- `인라인 코드`로 명령어, 파일명 표시
- 코드 블록에 언어 명시 (```bash, ```python 등)
- 테이블로 정보 정리
- 순서 있는/없는 리스트 활용

## 주의사항

- 기존 README.md가 있다면 먼저 읽고 내용을 보존/개선
- 프로젝트의 실제 파일 구조와 일치하도록 작성
- 실행 가능한 명령어만 포함 (테스트되지 않은 명령어 제외)
- 이모지는 사용자가 명시적으로 요청하지 않는 한 사용하지 않음
